# OLIMPIABIM 2026 - BIM AUTOMATION + IA

## Repositorio Oficial de Entregables

Bienvenido al repositorio oficial del concurso **BIM AUTOMATION + IA** de las **II Olimpiadas Internacionales BIM (OlimpiaBIM 2026)**, organizado por CEFIC, AECODE y UNSAAC.

Este repositorio es la plataforma de entrega de soluciones para todos los equipos participantes.

---

## Como Empezar (Participantes)

### Paso 1: Crear cuenta en GitHub
1. Ve a [github.com/signup](https://github.com/signup)
2. **IMPORTANTE**: Usa el **mismo correo electronico** con el que te inscribiste al concurso
3. Completa el registro y verifica tu correo

### Paso 2: Solicitar acceso al repositorio
1. Envia tu **nombre de usuario de GitHub** al correo del concurso: **aes.bim@unsaac.edu.pe**
2. Indica tu nombre completo y numero de equipo
3. Los organizadores te agregaran como colaborador
4. Acepta la invitacion que te llegara por correo

### Paso 3: Subir tus entregables
1. Entra a este repositorio desde tu navegador
2. Navega a la carpeta del ejercicio correspondiente (ej: `entregas > fase-1 > ronda-1 > ejercicio-1`)
3. Haz clic en **"Add file"** > **"Upload files"**
4. Arrastra tus archivos y haz clic en **"Commit changes"**

> Para instrucciones detalladas con imagenes, consulta la [Guia Completa para Participantes](GUIA-PARTICIPANTES.md)

## Estructura de Entregables

```
entregas/
  fase-1/
    ronda-1/
      ejercicio-1/     <-- Todos los equipos suben aqui su Ejercicio 1
      ejercicio-2/     <-- Todos los equipos suben aqui su Ejercicio 2
      ejercicio-3/     <-- Todos los equipos suben aqui su Ejercicio 3
    ronda-2/           <-- Caso grande (todos los equipos)
  fase-2/              <-- Entrega presencial (version final)
```

Cada equipo sube sus archivos directamente en la carpeta del ejercicio. Los organizadores pueden ver todas las entregas de todos los equipos en una sola carpeta por ejercicio.

---

## Nomenclatura Obligatoria de Archivos

### Fase 1 (por cada reto/ejercicio):
| Archivo | Formato |
|---|---|
| Codigo fuente | `BIMAUTO_NombreEquipo##_Codigo.zip` |
| README | `BIMAUTO_NombreEquipo##_README.pdf` |
| Outputs | `BIMAUTO_NombreEquipo##_Outputs.zip` |
| Dependencias | `BIMAUTO_NombreEquipo##_Dependencias.txt` (si aplica) |

### Fase 2:
| Archivo | Formato |
|---|---|
| Version final | `BIMAUTO_NombreEquipo##_VersionFinal.zip` |
| Logs y reportes | `BIMAUTO_NombreEquipo##_Logs_Reportes.pdf` |
| Presentacion | `BIMAUTO_NombreEquipo##_PresentacionFinal.pdf` |

> `##` = numero de equipo asignado (01, 02, 03...)

---

## Archivos Permitidos

| Tipo | Extensiones |
|---|---|
| Codigo y scripts | `.py`, `.cs`, `.dyn`, `.dyf`, `.json` |
| Documentacion | `.pdf`, `.md`, `.txt` |
| Comprimidos | `.zip` (outputs, evidencias, codigo) |
| Imagenes | `.png`, `.jpg` (capturas de pantalla) |
| Datos | `.csv` (datos tabulares de salida) |

### NO Permitidos
- Archivos `.rvt` (modelos Revit)
- Archivos mayores a 100 MB
- Ejecutables `.exe` o `.dll` sin codigo fuente
- Informacion personal (DNI, fotos carnet, etc.)

---

## Reglas del Repositorio

1. **Solo sube archivos a la carpeta del ejercicio correspondiente**
2. **No modifiques ni elimines** archivos de otros equipos
3. **Respeta la nomenclatura** obligatoria de archivos
4. **No subas credenciales**, API keys, ni informacion personal sensible
5. Cualquier manipulacion de entregas ajenas = **descalificacion inmediata**

---

## Uso de IA Generativa

El uso de IA (ChatGPT, Gemini, Claude, etc.) esta **permitido** con las siguientes condiciones:
- Incluir una **bitacora de uso de IA** con: prompts clave, que se uso, que fue verificado manualmente
- El equipo es **responsable de validar** todo resultado generado por IA
- La bitacora debe estar en el README de la entrega
